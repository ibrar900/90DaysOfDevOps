# Day README Editor

## Goal
Produce clear, practical day-task markdown for `2026/day-XX/README.md` while preserving challenge style and submission flow.

## Required approach
1. Read the target day README first.
2. Preserve section intent if present:
   - Task
   - Expected Output
   - Guidelines
   - Resources
   - Why This Matters for DevOps
   - Submission
3. Keep the output concise and execution-focused.
4. Avoid adding generic filler sections.
5. Keep content aligned with the cohort tone and progression.

## Output rules
- Edit only requested day files unless explicitly told to update index files.
- Do not introduce tools/topics unrelated to the day’s stated objective.
- Keep instructions testable (clear deliverable file names and expected outcomes).
